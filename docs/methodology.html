How We Rank Free AI Models

Every benchmark runs automatically at 03:00 UTC via GitHub Actions. No human grading, no sponsored rankings — just code and math.

Score Weights

Speed is never mixed into quality score. A model that writes perfect code but responds slowly still scores 100 on quality. Speed has its own separate leaderboard.

CategoryWeightWhat it measures
💻 Code30%Functions that actually run and produce correct output
🧠 Reasoning25%Logic, math, and puzzle problems with known correct answers
📋 Instructions15%Ability to follow exact formatting requirements
🌍 Translation10%Correct target language script and vocabulary
⚡ SpeedseparateTokens per second — own leaderboard, not in quality score
💻 Code Tests — Real Execution 30% of quality

The model is asked to write a Python function. We then actually run that function in a subprocess with test inputs and compare outputs to known correct values. Syntax errors, wrong logic, or empty responses all score 0. There's no partial credit for "it looks like it could work."

Example — is_prime test:

Prompt: "Write a Python function called is_prime(n) that returns True if n is prime..."

We run: is_prime(2) → True ✅ | is_prime(4) → False ✅ | is_prime(17) → True ✅

Score = passed / total test cases × 100
3 code problems tested: Prime checker · Fibonacci · Palindrome detection
🧠 Reasoning Tests — Verified Answers 25% of quality

Each question has exactly one correct answer. The model's response is checked against a list of accepted phrasings of that answer. We check only the first 50 characters to avoid giving credit for accidentally mentioning the answer later.

QuestionCorrect answer
All bloops are razzies, all razzies are lazzies — are all bloops lazzies?Yes
Train A: 120km/2h, Train B: 180km/3h — which is faster?Same (both 60 km/h)
Fox/chicken/grain river crossing — what to take first?Chicken
3 heads in a row — probability of 4th being heads?1/2
How many letters in MISSISSIPPI?11
📋 Instruction Following 15% of quality

Tests whether the model follows exact format requirements — critical for real-world API usage where you need structured output.

TaskHow graded
Return a JSON object with exactly 3 required keysJSON parsed, all keys present = 100, some missing = 50, invalid JSON = 0
List exactly 5 programming languages, numbered 1-5Count numbered lines: exact = 100, any number present = 50, none = 0
Describe Paris in exactly 3 sentencesSentence count: exact = 100, ±1 = 50, off by more = 0
🌍 Translation Tests 10% of quality

Lightweight script and vocabulary detection. We don't measure translation quality deeply — that would require human evaluation. We check that the model translated into the right language at all.

TaskCheck
English → RussianCyrillic character ratio > 20%

⚡ Speed Measurement separate leaderboard

We send 3 prompts of varying length (short, medium, long) and measure tokens per second from the API response's usage field. The fastest model in a given run gets a normalised score of 100; others are scaled proportionally.

⚠️ Speed depends heavily on server load at the time of testing. Numbers reflect real-world performance at 03:00 UTC but may differ at peak hours.

📐 Size Categories

Comparing a 4B model against a 120B model in a single ranking is misleading. We group models by active parameter count and rank within groups.

CategoryParameter rangeExample models
🟢 Small≤ 10BLlama 3.1 8B, Qwen 3 4B
🟡 Medium10B – 50BLlama 4 Scout 17B, Mistral Small 24B, Qwen 3 32B
🔴 Large50B+Llama 3.3 70B, GPT-OSS 120B, Nemotron 120B

For MoE models (e.g. Llama 4 Scout 17Bx16E), we use the per-expert active parameter count as the size for categorisation since that reflects actual compute at inference time.
